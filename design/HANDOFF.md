# 포켓몬 스피드 퀴즈 앱 — 디자인 핸드오프

> 이 문서는 디자인 시안(목업)을 실제 앱으로 구현하기 위한 사양서입니다.
> 모든 화면 스크린샷은 `screens/` 폴더에 있습니다. 원본 인터랙티브 시안은
> `포켓몬 스피드 퀴즈.dc.html` 한 파일이며, 브라우저에서 바로 열어 동작을 확인할 수 있습니다.

## 0. 한 줄 요약

내가 등록한 6마리 엔트리와 현재 상위 메타 포켓몬의 **스피드(실효 속도)를 좌/우 스와입으로 비교**하는
모바일 퀴즈 앱. 난이도가 올라갈수록 아이템·랭크·날씨·특성·노력치/성격·상태이상 변수가 누적됨.

- **타깃 플랫폼**: 모바일 (세로 390×844 기준, 반응형)
- **프론트엔드**: React + Vite + TypeScript, 디자인 시스템 **HeroUI v3**, Vercel 배포
- **문제 생성**: 모델(LLM)로 실시간 생성, 출제는 **Y/N(예/아니오) 단일 형식**, 정답지·해설지 동시 생성
- **테마**: 라이트 / 다크 둘 다 지원

> 이 문서는 **디자인 + 프론트엔드 + 퀴즈/공식 로직**만 다룹니다. 서버·데이터 저장(DynamoDB 등)은 별도 설계.

## 3. 화면별 구현 사양

### 3.1 엔트리 목록 (`01`, 다크 `05`)
- 헤더: "내 엔트리" / 부제 "최대 6마리".
- **표시 스탯 필터** (헤더 아래 가로 스크롤 칩): `스피드(기본) · HP · 공격 · 방어 · 특공 · 특방`.
  선택한 스탯이 각 카드 하단 큰 숫자로 바뀜. 기본 선택 = 스피드.
- 2열 그리드 카드: 우상단 `Lv` 배지, 가운데 스프라이트, 이름, 타입 칩 2개, 하단에 `{선택 스탯} : {값}`.
- 6번째 슬롯이 비면 점선 "포켓몬 추가" 카드 (탭 → 포켓몬 만들기).
- 하단 고정 CTA: "퀴즈 시작하기" → 난이도 선택.

### 3.2 난이도 선택 (`02`)
4단계. 각 카드에 포함 변수를 칩으로 표기. **한 세션 = 하나의 난이도** (중요: 세션 내 난이도 혼합 없음).

| 단계 | 이름 | 누적되는 변수 |
|---|---|---|
| 1 | 쉬움 | 베이스 종족값 속도만 (speed tier 비교) |
| 2 | 보통 | + 지닌 물건(구애스카프 등), 랭크 업/다운 |
| 3 | 어려움 | + 날씨, 특성, 노력치/성격 (메타 대중 수치) |
| 4 | 마스터 | + 상태이상(마비 등) 전체 변수 |

- 하단 CTA: "{선택 난이도}으로 시작".

### 3.3 스피드 스와입 퀴즈 (`03`, 다크 `06`) — 핵심 화면
- 상단: 난이도 배지 + 진행도 `n / 총문제`.
- 질문 텍스트 (예: "내 토게키스가 무쇠손보다 빠를까?"). **속도 실제 수치는 숨김** (`?`로 표시).
- 카드 높이 ≈ 화면의 2/3, 세로 중앙. 뒤에 다음 카드가 살짝 비침.
- 카드 내부: MY(왼쪽, 파랑) vs META(오른쪽, 빨강), 가운데 VS 뱃지. 각 포켓몬 아래 `?` 속도 자리.
- **적용 변수 칩**: 카드 하단에 이번 문제에 적용된 변수 노출 (예: "쾌청 · 엽록소 ×2", "구애스카프 ×1.5", "마비 ÷2", "속도 ▼ 1랭크"). 난이도에 따라 노출 변수 종류가 달라짐.
- **인터랙션**: 카드를 좌우 드래그(또는 ✕/✓ 영역 탭).
  - 우(→) = **예(YES)** / 좌(←) = **아니오(NO)**.
  - 드래그 임계값 100px 초과 시 확정. 드래그 양에 따라 "예 ✓" / "아니오 ✕" 스탬프 오버레이 페이드인, 카드 회전(`rotate(dx*0.04deg)`).
  - 확정 시 정답이면 초록, 오답이면 빨강 풀스크린 오버레이 + 한 줄 해설을 잠깐 표시 → 다음 문제.
- 하단: `← 아니오 · 예 →` 힌트.

### 3.4 포켓몬 만들기 / 편집 (`04`)
엔트리 1마리를 정의·저장하는 폼 (세로 스크롤). 원본 스크린샷 레이아웃을 따름.
- 스프라이트 + 포켓몬 검색 셀렉트 + `Lv`(기본 50).
- 타입 칩(자동 표시).
- 기술 4칸 셀렉트.
- 특성 / 지닌 물건 / 성격 셀렉트.
- **스탯 테이블**: 행 = HP/공격/방어/특공/특방/스피드. 열 = `기본(종족값) · 노력치(EV 슬라이더, -/+) · 성격 배율 · 합계`. 상단에 "남은 노력치 n/총합" 표시(챔피온스 룰 EV 총량).
- CTA: "엔트리에 저장" (DynamoDB 엔트리 테이블에 upsert).

### 3.5 퀴즈 완료 (`07`) — 듀오링고풍
- **떨어지는 콘페티** 애니메이션 (≈26조각, 6색, `confFall` 키프레임으로 무한 낙하).
- **완료 아이콘 = 내 엔트리 중 랜덤 1마리의 움직이는 스프라이트(GIF)**. `bob` 모션으로 통통 튀고 아래 그림자가 호흡(`shadowPulse`). 애니메이션 GIF 없는 포켓몬은 official-artwork로 폴백.
  - (참고: 정적 스크린샷에선 GIF가 안 잡혀 정적 이미지로 캡처됨. 실제 앱에선 움직임.)
- 제목 "퀴즈 완료!" + 부제 "{포켓몬 이름}와 함께 완주했어요".
- 큰 점수 `정답수 / 총문제`.
- 통계 카드 3개: 정답 / 오답 / 정답률(%).
- **완료 효과음**: 화면 진입 시 상승 아르페지오 차임 자동 재생 (Web Audio, C-E-G-C 트라이앵글 오실레이터). 시안에는 "🔊 완료 효과음 듣기" 버튼으로 수동 재생 (브라우저 자동재생 정책 때문). 실제 앱은 화면 전환 시 자동 재생 권장.
- CTA: "정답 & 해설 보기".

**완료 차임 구현 (그대로 이식 가능)**:
```js
function playChime() {
  const ctx = new (window.AudioContext || window.webkitAudioContext)();
  [523.25, 659.25, 783.99, 1046.5].forEach((f, i) => {
    const o = ctx.createOscillator(), g = ctx.createGain();
    o.type = 'triangle'; o.frequency.value = f;
    o.connect(g); g.connect(ctx.destination);
    const t = ctx.currentTime + i * 0.11;
    g.gain.setValueAtTime(0, t);
    g.gain.linearRampToValueAtTime(0.28, t + 0.02);
    g.gain.exponentialRampToValueAtTime(0.0008, t + 0.32);
    o.start(t); o.stop(t + 0.34);
  });
}
```

  - 질문 텍스트.
  - MY vs META 미니 매치업 (스프라이트 + 이름).

## 4. 퀴즈 생성 & 스피드 공식 사양

### 4.1 난이도별 적용 변수 (누적)
| 난이도 | 종족값 | 노력치/성격 | 아이템 | 랭크 | 날씨 | 특성 | 상태이상 |
|---|---|---|---|---|---|---|---|
| 쉬움 | ✅ | — | — | — | — | — | — |
| 보통 | ✅ | — | ✅ | ✅ | — | — | — |
| 어려움 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | — |
| 마스터 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

> 쉬움은 **베이스 종족값 속도(speed tier)** 만 비교. 윗단계로 갈수록 변수를 조금씩 섞음.

### 4.2 실효 속도 공식 (Pokémon Champions 룰)
> ⚠️ 아래는 일반적인 본가 룰 기준 골격입니다. **Pokémon Champions의 정확한 계수/반올림 규칙은
> 4.4의 검증 프롬프트로 반드시 확인 후 확정하세요.** 시안의 해설 수치는 예시값입니다.

1. **스탯 산출** (Lv50 기준):
   `스피드 = floor( (floor((2*종족값 + 개체값 + floor(EV/4)) * Lv/100) + 5) * 성격배율 )`
   - 개체값(IV) 기본 31, 성격배율 = 스피드 ↑성격 1.1 / ↓성격 0.9 / 무관 1.0.
2. **랭크 보정**: +단계 `(2+n)/2`, -단계 `2/(2+n)` (n=1..6).
3. **아이템**: 구애스카프 ×1.5, 게으름의징표류 등은 룰 확인.
4. **특성**: 엽록소(쾌청)/세이비(우천)/모래헤치기(모래)/눈치우기(설경) = ×2, 곡예 등 조건부.
5. **날씨**: 위 특성과 연동.
6. **상태이상**: 마비 = 본가 ÷2 (단, **세대/룰에 따라 ÷2가 아닐 수 있음 — 검증 필수**).

적용 순서(권장): 종족값→EV/성격으로 기본 스피드 → 랭크 → 아이템/특성 배율 → 상태이상.

### 4.3 문제 규격 (모델 출력 JSON 스키마)
출제는 **Y/N 단일 형식**. 모델이 문제+정답+해설을 한 번에 생성:

```jsonc
{
  "difficulty": "보통",
  "my": { "dex_id": 727, "name_ko": "어흥염", "effective_speed": 90, "factors": ["구애스카프 ×1.5"] },
  "opp": { "dex_id": 445, "name_ko": "한카리아스", "effective_speed": 102, "factors": [] },
  "question": "내 어흥염이 한카리아스보다 빠를까?",
  "applied_chips": [ {"label": "구애스카프 ×1.5", "color": "#006FEE"} ], // 카드에 노출(속도는 숨김)
  "answer": false,  // 예/아니오 → boolean (YES=true)
  "explanation": "어흥염 60×1.5=90 < 한카리아스 102 → 아니오"
}
```
- `effective_speed`는 **클라이언트에 숨김**(UI는 `?`). 정답·해설 채점·표시에만 사용.
- `applied_chips`는 난이도에 맞는 변수만 포함.

### 4.4 공식 검증 프롬프트 (모델에게 "공식이 맞는지" 확인시키는 용도)
```
너는 Pokémon Champions 룰의 스피드/결정력(실효 속도) 계산 검수기다.
입력으로 두 포켓몬의 {종족값, 개체값, 노력치, 성격, 레벨, 아이템, 특성, 날씨,
랭크 변화, 상태이상}이 주어진다.

1) 각 포켓몬의 실효 스피드를 단계별로 계산하라:
   (a) Lv50 기준 기본 스피드 산출식과 반올림(floor) 위치를 명시
   (b) 성격 배율(1.1/0.9/1.0) 적용
   (c) 랭크 보정 (+n: (2+n)/2, -n: 2/(2+n))
   (d) 아이템·특성·날씨 배율
   (e) 상태이상(예: 마비) 보정 — Pokémon Champions에서의 정확한 계수를 적용
2) 각 단계의 중간값을 보여주고, 최종 실효 스피드를 비교해 더 빠른 쪽을 판정하라.
3) 사용한 모든 계수/반올림 규칙이 Pokémon Champions 공식 룰과 일치하는지 점검하고,
   본가 9세대와 다른 점이 있으면 명확히 지적하라.
4) 출력은 위 4.3 JSON 스키마(question/answer/explanation/applied_chips)로 반환하라.
   - answer는 "내 포켓몬이 더 빠른가?"에 대한 boolean.
   - explanation은 핵심 계산식 한 줄(예: "어흥염 60×1.5=90 < 한카리아스 102 → 아니오").
모르거나 룰이 불확실한 계수는 추측하지 말고 "검증 필요"로 표시하라.
```

src/
  routes/
    EntryListPage.tsx      // 화면 1 (+ 스탯 필터 상태)
